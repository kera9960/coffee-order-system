# Git Guide

이 문서는 에이전트가 Git을 사용할 때 지켜야 하는 규칙을 정의한다.

Git 이력 변경과 원격 저장소 작업은 사용자가 직접 수행하는 것을 기본으로 한다.

## 기본 원칙

- 에이전트는 기존 사용자 변경사항을 보존한다.
- 현재 상태를 확인하지 않고 파일을 되돌리지 않는다.
- commit, push, merge는 수행하지 않는다.
- Git 이력을 변경하는 명령을 실행하지 않는다.
- 필요한 Git 명령, 브랜치 이름과 커밋 메시지는 추천할 수 있다.

## 허용되는 작업

프로젝트 상태를 확인하기 위한 읽기 전용 명령은 실행할 수 있다.

```bash
git status
git diff
git diff --staged
git log
git show
git branch
git branch --show-current
```

읽기 작업은 다음 목적으로만 사용한다.

- 현재 브랜치 확인
- 기존 사용자 변경사항 확인
- 이번 작업의 변경 범위 확인
- 승인되지 않은 변경 여부 확인
- Evaluate 결과 작성

## 수행하지 않는 작업

다음 작업은 사용자가 직접 수행한다.

- 브랜치 생성, 전환과 삭제
- commit, push, merge와 rebase
- 태그 생성과 삭제
- Pull Request 생성과 병합
- 원격 저장소 변경
- Git 이력 수정

에이전트는 필요한 경우 실행 방법만 안내한다.

## 위험한 명령

다음 명령은 작업 내용이나 Git 이력을 손상할 수 있으므로 직접 실행하지 않는다.

```bash
git reset --hard
git clean -fd
git clean -fdx
git restore .
git checkout -- .
git branch -D
git push --force
git push --force-with-lease
```

사용자가 위 명령의 사용 방법을 요청하면 다음 내용을 먼저 설명한다.

- 삭제되거나 되돌아가는 내용
- 복구 가능 여부
- 기존 변경사항을 보존하는 방법
- 더 안전한 대안

## 기존 변경사항 보호

작업 시작 전 `git status`와 필요한 경우 `git diff`를 확인한다.

- 기존 변경을 에이전트 작업으로 간주하지 않는다.
- 기존 변경을 임의로 포맷하거나 정리하지 않는다.
- 기존 변경을 되돌리지 않는다.
- 같은 파일을 수정할 때 기존 내용을 보존한다.
- 충돌 가능성이 있으면 Plan에 보고한다.
- 변경 주체를 구분할 수 없으면 추측하지 않고 사실대로 보고한다.

## 작업 완료 지원

Evaluate가 끝난 뒤 다음 정보를 제공할 수 있다.

- 변경된 파일 목록
- 작업 내용 요약
- 검증 결과
- 추천 브랜치 이름
- 추천 커밋 메시지
- PR 설명 초안

추천 내용은 실제 변경사항과 프로젝트 규칙을 기준으로 작성한다.

단, 실제 commit, push, merge는 수행하지 않는다.

## 금지 사항

- 사용자 변경사항 초기화
- 승인되지 않은 Git 이력 변경
- 강제 푸시와 원격 브랜치 삭제
- 사용자가 작성한 커밋 수정
- 실패한 검증 결과 은폐
- `.git` 내부 파일 직접 수정
