# Evaluate Guide

## 목적

Evaluate는 Generate에서 만든 변경사항이
승인된 Plan과 완료 조건을 충족하는지 검증하는 단계다.

Evaluate에서는 파일을 변경하지 않는다.

## 검증 항목

다음 내용을 확인한다.

- 승인된 Plan 범위만 변경했는가
- 비즈니스 정책을 충족하는가
- API와 DB 문서가 구현과 일치하는가
- 정상 동작과 주요 실패 조건을 테스트했는가
- 기존 기능에 예상하지 못한 영향이 없는가
- 사용자의 기존 변경사항을 보존했는가
- 임시 코드와 불필요한 변경이 남아 있지 않은가

## 변경사항 확인

`git status`와 `git diff`를 사용해 변경사항을 확인한다.

다음 내용을 구분하여 보고한다.

- 이번 작업에서 변경한 파일
- 사용자가 기존에 수정한 파일
- 승인된 범위를 벗어난 변경
- 확인이 필요한 변경

변경 주체를 확실히 구분할 수 없다면 추측하지 않고 그대로 보고한다.

## 검증 명령어

프로젝트 상태와 작업 범위에 맞는 명령을 실행한다.

```bash
./gradlew compileJava
./gradlew test
./gradlew build
```

모든 명령을 반드시 실행해야 하는 것은 아니다.

일부 명령을 실행하지 않았다면 다음 내용을 기록한다.

- 실행하지 않은 명령
- 실행하지 않은 이유
- 그로 인해 확인하지 못한 내용

실행하지 않은 테스트를 통과했다고 보고하지 않는다.

## 실패 처리

검증에 실패해도 Evaluate 단계에서 파일을 수정하지 않는다.

- 승인된 범위 안의 구현 문제라면 Generate 단계로 돌아간다.
- 정책, API, DB 또는 작업 범위 변경이 필요하면 Plan 단계로 돌아간다.
- 환경 문제로 실행하지 못했다면 원인과 확인하지 못한 내용을 보고한다.
- 테스트를 삭제하거나 검증을 약화하여 성공으로 만들지 않는다.

## 결과 형식

```text
## 변경 요약

## 변경 파일

## 검증 결과
- 컴파일:
- 테스트:
- 빌드:

## 완료 조건 확인

## 문서 일치 여부

## 발견된 문제

## 실행하지 못한 검증

## 최종 판단
```

## 최종 판단

최종 결과는 다음 중 하나로 작성한다.

- `완료`: 승인된 Plan과 완료 조건을 충족하고 필요한 검증을 통과함
- `Generate로 복귀`: 승인된 범위 안에서 수정이 필요함
- `Plan으로 복귀`: 정책이나 작업 범위의 재결정이 필요함
- `검증 불완전`: 환경 문제 등으로 일부 검증을 수행하지 못함

문제가 남아 있거나 검증하지 못한 항목이 있다면
완료했다고 보고하지 않는다.

## 금지 사항

- 코드, 테스트, 문서와 설정 변경
- 테스트 삭제 또는 비활성화
- 실패 결과 은폐
- 실행하지 않은 검증을 성공으로 보고
- 사용자 변경사항 되돌리기
- commit, push, merge
