# Policy — 비즈니스 정책

이 폴더는 커피 주문 시스템의 여러 기능에서 공통으로 사용하는
비즈니스 규칙을 관리한다.

정책 문서는 비즈니스 규칙의 진실의 원천(SSOT)이다.
각 기능의 `design.md`, API 명세, DB 명세는 정책을 중복해서 정의하지 않고
이 폴더의 문서를 참조한다.

## 문서 역할

- `policy`: 시스템이 지켜야 하는 비즈니스 규칙
- `design.md`: 정책을 기능에서 어떻게 구현하는지 설명
- `docs/api/README.md`: 정책을 외부 API 계약으로 표현
- `docs/db/README.md`: 정책을 데이터 구조와 제약조건으로 표현
- `ADR`: 중요한 설계 방법을 선택한 이유를 기록

정책과 다른 문서의 내용이 충돌하면 정책 문서를 우선 확인한다.

## 정책 목록

| 정책 | 문서 | 주요 적용 기능 |
| --- | --- | --- |
| 포인트 정책 | `docs/policy/point.md` | 포인트 충전, 주문 결제, 거래 이력 |
| 메뉴 판매 정책 | `docs/policy/menu-sales.md` | 메뉴 목록, 신규 주문, 인기 메뉴 |
| 주문 정책 | `docs/policy/order.md` | 주문 생성, 주문 목록, 주문 상세 |
| 인기 메뉴 정책 | `docs/policy/popular-menu.md` | 인기 메뉴 집계 및 조회 |

## 정책 변경 원칙

- 같은 정책을 여러 문서에 복사하지 않는다.
- 정책이 변경되면 해당 정책을 참조하는 기능의 영향을 검토한다.
- 정책 변경으로 API, DB, 코드 수정이 필요한지 함께 확인한다.
- 정책에 없는 내용을 구현 과정에서 임의로 결정하지 않는다.
- 기술적인 구현 방법은 해당 기능의 `design.md` 또는 ADR에 기록한다.
