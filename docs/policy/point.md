# 포인트 정책

## 규칙

### 포인트 충전

- 포인트는 1원당 1포인트로 충전한다.
- 예를 들어 5,000원을 충전하면 5,000포인트가 증가한다.
- 충전 금액은 0보다 커야 한다.
- 충전 거래는 고객의 포인트 잔액을 증가시킨다.
- 충전 거래는 특정 주문과 연결되지 않는다.

### 포인트 결제

- 커피 주문의 결제 수단은 포인트만 허용한다.
- 주문 금액 1원당 1포인트를 차감한다.
- 예를 들어 5,000원인 주문은 5,000포인트를 사용한다.
- 고객의 포인트가 주문 금액보다 부족하면 주문할 수 없다.
- 포인트 잔액은 음수가 될 수 없다.
- 포인트 사용 거래는 결제한 주문과 연결한다.

### 포인트 거래 이력

- 모든 포인트 충전과 사용은 거래 이력으로 기록한다.
- 충전 거래의 포인트 증감량은 양수로 기록한다.
- 사용 거래의 포인트 증감량은 음수로 기록한다.
- 거래 이력에는 거래 후 포인트 잔액을 기록한다.
- 거래 금액은 원화 금액이 아니라 실제 포인트 증감량을 의미한다.

### 데이터 일관성

- 포인트 충전과 충전 이력 기록은 모두 성공하거나 모두 실패해야 한다.
- 주문 생성, 포인트 차감, 포인트 사용 이력 기록은 모두 성공하거나 모두 실패해야 한다.
- 포인트 잔액과 거래 이력이 서로 다른 결과로 저장되어서는 안 된다.

### 현재 범위

- 현재 범위에는 포인트 환불 기능을 포함하지 않는다.
- 주문 취소와 포인트 환불 기능이 추가되면 별도의 정책을 정의한다.

## 근거 / 배경

커피 주문의 결제 과정을 단순화하기 위해 포인트를 유일한 결제 수단으로 사용한다.

포인트 환산 비율은 1원당 1포인트로 고정되어 있으므로
별도의 환산 비율을 관리하지 않는다.

현재 포인트 잔액과 거래 이력을 함께 관리하여
주문 시 잔액을 빠르게 확인하면서 잔액 변경 근거도 추적할 수 있도록 한다.

## 적용 대상

- 포인트 충전
- 포인트 잔액 조회
- 포인트 거래 이력 조회
- 주문 생성 및 결제
- 주문 상세 조회
