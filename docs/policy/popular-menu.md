# 인기 메뉴 정책

## 규칙

### 집계 기간

- 인기 메뉴는 조회 시점을 기준으로 최근 7일 동안의 주문을 집계한다.
- 최근 7일 이전의 주문 건수는 현재 인기 순위에 합산하지 않는다.

### 집계 대상 주문

- 주문 상태가 `COMPLETED`인 주문만 집계한다.
- 주문 상태가 `CANCELED`인 주문은 집계하지 않는다.
- 최근 7일 동안 생성된 주문만 사용한다.

### 집계 대상 메뉴

- 현재 판매 상태가 `ON_SALE`인 메뉴만 인기 메뉴 후보가 된다.
- `STOPPED` 메뉴는 해당 기간의 주문 건수가 많더라도 인기 메뉴 결과에서 제외한다.
- 판매 중인 메뉴를 먼저 선별한 뒤 인기 순위를 계산한다.
- 전체 순위를 먼저 계산하고 판매 중지 메뉴를 나중에 제거하지 않는다.

### 집계 기준

- 인기 메뉴는 해당 메뉴가 포함된 주문 건수를 기준으로 계산한다.
- 메뉴의 총 판매 수량은 인기 순위에 반영하지 않는다.
- 한 주문에서 같은 메뉴를 여러 개 주문해도 주문 건수는 1건으로 계산한다.
- 한 주문에 서로 다른 여러 메뉴가 포함되면 각 메뉴의 주문 건수를 각각 1건씩 증가시킨다.

### 반환 개수

- 인기 메뉴는 최대 3개를 반환한다.
- 조건을 만족하는 메뉴가 3개보다 적으면 존재하는 메뉴만 반환한다.
- 조건을 만족하는 메뉴가 없으면 빈 목록을 반환한다.
- 반환 개수를 채우기 위해 판매 중지 메뉴나 주문 이력이 없는 메뉴를 포함하지 않는다.

### 정렬 기준

인기 메뉴는 다음 순서로 정렬한다.

1. 최근 7일 동안 해당 메뉴가 포함된 완료 주문 건수 내림차순
2. 주문 건수가 같으면 해당 메뉴가 포함된 가장 최근 완료 주문 시각 내림차순
3. 가장 최근 주문 시각까지 같으면 메뉴 ID 오름차순

가장 최근 주문 시각은 메뉴 생성일이나 수정일이 아니라,
해당 메뉴가 포함된 완료 주문 중 가장 최근 주문의 주문 시각을 의미한다.

## 근거 / 배경

최근 주문 데이터를 기준으로 고객 반응이 인기 메뉴 순위에 반영되도록 한다.

한 주문에서 많은 수량을 구매한 경우가 순위에 지나친 영향을 주지 않도록
판매 수량이 아니라 해당 메뉴가 포함된 주문 건수를 집계 기준으로 사용한다.

취소된 주문은 정상적으로 완료된 판매로 보지 않으므로 집계에서 제외한다.

- 인기 메뉴 정렬
- 향후 인기 메뉴 캐시 또는 통계 기능
