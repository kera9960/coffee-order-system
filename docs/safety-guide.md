# Safety Guide

이 문서는 데이터 손실, 보안 문제 또는 프로젝트 전체에 큰 영향을 줄 수 있는 작업의 처리 기준을 정의한다.

위험 작업은 영향과 대안을 먼저 설명하고,
사용자의 명시적인 승인을 받은 뒤 수행한다.

## 위험 작업 기준

다음 중 하나에 해당하면 위험 작업으로 판단한다.

- 데이터가 삭제되거나 손상될 수 있음
- 기존 기능이나 프로젝트 구조가 크게 변경됨
- 프로젝트 실행 또는 배포에 영향을 줄 수 있음
- 보안 수준이 낮아질 수 있음
- 운영 환경이나 외부 사용자에게 영향을 줌
- 비용이 발생할 수 있음
- 변경을 되돌리기 어려움
- 사용자의 기존 작업이 손상될 수 있음

## 승인받아야 하는 작업

### 데이터베이스

- 테이블, 컬럼, 관계 변경
- 제약조건과 인덱스 변경
- 마이그레이션 실행
- 데이터 생성, 수정, 삭제와 초기화
- 운영 DB 접근
- 대량 데이터 변경

### 설정과 의존성

- 애플리케이션, Docker와 배포 설정 변경
- DB, Redis, Kafka와 외부 시스템 연결 설정 변경
- 라이브러리와 플러그인 추가, 삭제 또는 버전 변경
- Java 또는 Spring Boot 버전 변경

### 보안

- 인증·인가 방식 변경
- Spring Security와 JWT 설정 변경
- CORS와 CSRF 설정 변경
- 공개 API 범위 변경
- 보안 검증 완화 또는 우회

### 삭제와 구조 변경

- 클래스, API, 기능과 테스트 삭제
- 여러 파일의 일괄 이동
- 패키지 구조 변경
- 기존 기능 제거

### 외부 시스템

- 이메일, 문자와 알림 발송
- 결제 요청
- 외부 API의 데이터 생성·변경
- 클라우드 자원 생성·삭제
- 운영 서버 배포 또는 종료
- 비용이 발생할 수 있는 호출

가능한 경우 운영 환경 대신 mock, stub 또는 테스트 환경을 사용한다.

## Plan에 포함할 내용

위험 작업이 필요하면 다음 내용을 Plan에 작성한다.

```text
## 위험 작업

작업:
- 수행하려는 변경

필요한 이유:
- 변경이 필요한 이유

영향:
- 영향을 받는 코드, 문서, 데이터와 환경

위험:
- 실패하거나 잘못 수행했을 때 발생할 수 있는 문제

대안:
- 위험을 줄이거나 변경을 피할 수 있는 방법

롤백:
- 변경을 되돌리는 방법

검증:
- 변경 후 확인할 테스트와 명령
```

사용자 승인 전에는 해당 작업을 수행하지 않는다.

작업 중 승인되지 않은 새로운 위험이 발견되면
즉시 중단하고 Plan으로 돌아간다.

## 비밀정보 처리

다음 정보는 코드, 테스트, 로그와 문서에 직접 작성하지 않는다.

- 비밀번호
- API 키
- JWT 비밀키
- 액세스 토큰
- DB 운영 계정
- 클라우드 자격증명
- 개인 인증정보

비밀정보는 환경변수 또는 프로젝트에서 정한 안전한 방식으로 주입한다.

비밀값의 실제 내용을 출력하거나 복사하지 않는다.

## 즉시 중단 조건

다음 상황이 발견되면 작업을 중단하고 사용자에게 보고한다.

- 운영 환경으로 보이는 연결 정보 발견
- 실제 데이터가 삭제될 가능성
- 저장소에 비밀정보가 노출됨
- 승인 범위를 크게 벗어나는 변경 필요
- 사용자의 기존 작업이 손상될 가능성
- 복구 방법이 불명확함
- 외부 비용이 발생할 가능성
- 보안 수준을 낮춰야만 진행 가능

## 금지 사항

- 승인되지 않은 위험 작업 수행
- 테스트에 운영 데이터 사용
- 비밀정보를 저장소에 기록
- 테스트 통과를 위한 보안 검증 우회
- 테스트 실패를 해결하기 위한 기능 또는 테스트 삭제
- 위험을 숨기거나 축소하여 보고
- 복구 방법을 확인하지 않은 파괴적 작업
